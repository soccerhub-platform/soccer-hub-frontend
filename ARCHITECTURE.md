# Архитектура Soccer Hub Frontend

Документ фиксирует текущее состояние фронтенда и MVP-фокус: провести клиента по цепочке  
`лид → пробная → клиент → договор → оплата → посещаемость → продление`.

## Технологический стек

- React 18 + TypeScript
- React Router 6
- Tailwind CSS
- Vite + Vitest

## Структура каталогов

```text
src/
  modules/
    dispatcher/   # super admin / диспетчер
    admin/        # админ филиала
    coach/        # тренер
  shared/         # auth, api, ui, типы, общие утилиты
  App.tsx         # верхнеуровневый роутинг
```

## Роли и зоны ответственности

- **Dispatcher (super admin):**
  - лиды верхнего уровня (`/dispatcher/leads`);
  - клубы/филиалы (`/dispatcher/clubs`);
  - создание администраторов филиалов и назначение в филиалы (`/dispatcher/admins`).
- **Admin (админ филиала):**
  - воронка филиала (`/admin/leads`);
  - группы/тренеры/контракты/платежи (`/admin/groups`, `/admin/coaches`, `/admin/contracts`, `/admin/payments`);
  - операционное управление пользователями филиала (`/admin/users`).
- **Coach (тренер):**
  - расписание, занятия, отметка посещаемости и отчеты (`/coach/*`).

## Маршрутизация

Корневой роутер:

- `/login`
- `/dispatcher/*`
- `/admin/*`
- `/coach/*`

### Важные рабочие маршруты (MVP P0)

- Dispatcher: `/dispatcher/clients`, `/dispatcher/trial-trainings`, `/dispatcher/leads`, `/dispatcher/admins`
- Admin: `/admin/payments`, `/admin/users`, `/admin/contracts`, `/admin/leads`

Эти маршруты должны быть доступны не только по URL, но и из бокового меню соответствующей роли.

## MVP flow и инварианты состояний

Фронтенд поддерживает строгое прохождение по этапам:

1. Создание/квалификация лида.
2. Назначение и фиксация результата пробной.
3. Конвертация лида в клиента с созданием договора и назначением в группу.
4. Переход в ожидание оплаты и фиксация платежа.
5. Дальнейшая работа с посещаемостью и продлениями.

Ключевой принцип MVP: закрываем денежный цикл и операционную дисциплину вручную в системе, интеграции — следующим этапом.
